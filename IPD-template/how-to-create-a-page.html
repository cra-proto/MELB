How to create a page

A semantic model for CRA web content authors. Use this guide to decide what information belongs on a page based on what the page is trying to help a user do.

Most CRA service pages answer one question: How do I do this thing? Whether the thing is applying for a benefit, filing a return, registering for a program, or claiming a credit, the underlying structure is the same. The sections below form a complete content model. Not every page needs every section, but authors should consciously decide to include or exclude each one.

Sections

Choosing a page format
Decide whether your content works as a single page or needs the multi-page subway pattern
Title
Write a clear, action-oriented title that sets user expectations
Introductory text
Tell the user whether they are in the right place with a brief doormat description
Before you start
List what the user needs to have ready before they begin
Eligibility
Help users determine whether they can use the service, apply for the benefit, or claim the credit
Step-by-step instructions
Walk the user through the process of doing the thing, in chronological numbered steps
After you submit
Tell the user what to do or expect immediately after completing their submission
What happens next
Explain the CRA's side of the process so the user understands what is happening
Processing time
Set clear expectations about how long the user should wait for a result
How you will be notified
Tell the user how and where they will receive the CRA's response
If you do not get a response
Give the user a clear path when the expected timeframe has passed
How to check status
Show the user where and how to track the progress of their submission
If something goes wrong
Cover error recovery when a submission is rejected, denied, or needs correction
If there is a mistake or something changes
Tell users how to correct errors or report changes in circumstances after submitting
Contact information
Provide clear, complete contact details with phone numbers, hours, and self-serve options
