How the changelog system works

The “What’s new” page is driven by a single JSON file. Adding a changelog entry is one of the simplest tasks in the repo.

On this page

  • How it works
  • JSON entry structure
  • How to add an entry

How it works

The changes page (/en/changes/index.html) uses WET-BOEW’s wb-jsonmanager plugin and data-wb-json attributes to dynamically load and render entries from assets/files/changes/whats_new_ucdg.json.

The home page also reads this JSON to display the date of the most recent update.

JSON entry structure

Each entry in the JSON array has these bilingual fields:

JSON changelog entry fields
FieldExamplePurpose
title_en / title_fr“Subway navigation pattern”Entry headline
description_en / description_fr“Added guidance for the subway...”Short description
linkText_en / linkText_fr“View the subway pattern”Link label
linkUrl_en / linkUrl_fr“/design/en/design-systems/...”Link destination (absolute from /design/)
date_en / date_fr“January 30, 2024”Human-readable date
modified_en / modified_fr“New” or “Updated”Change type badge
keywords_en / keywords_fr“design-system”Filter category: design-system, writing, process-activity, or other

How to add an entry

  1. Open the JSON file

    Edit assets/files/changes/whats_new_ucdg.json.

  2. Add a new object at the beginning of the array

    The most recent entry should be first. Copy an existing entry and update all fields.

  3. Fill in both languages

    Every field has an _en and _fr version. Both must be filled in.

  4. Commit and push

    The changes page and home page will update automatically.
