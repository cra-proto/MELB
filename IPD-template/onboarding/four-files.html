The four files everyone must know

Before you edit anything in the repo, understand these four files. They power every page on the site.

On this page

  • 1. ucdg.js — the site engine
  • 2. menu-include-en.html — the navigation
  • 3. footer-include-en.html — the footer
  • 4. ucdg-theme.css — the custom theme

1. assets/js/ucdg.js — the site engine

This is the most important file in the entire repo. It runs on every page and does two things:

  1. Detects the page language

    It reads the lang attribute from the <html> tag to determine whether to load English or French includes.

  2. Injects the menu and footer

    It uses XMLHttpRequest to fetch two HTML fragment files and injects them into placeholder <div> elements on the page:

    • <div id="menu-include"></div> → receives the navigation menu
    • <div id="footer-include"></div> → receives the footer

Critical path dependency: The include paths in ucdg.js are absolute, starting with /design/. If the repo were ever renamed or moved to a different GitHub Pages path, the menu and footer would stop loading on every page.

The file also handles mobile menu overlay behaviour using jQuery and WET-BOEW overlay events, and pre-opens all <details> elements inside the mobile menu panel.

2. assets/includes/menu-include-en.html — the navigation

This HTML fragment contains the entire site navigation in two versions:

Desktop
A horizontal wb-menu bar with the site title on the left and topic links on the right: Writing, Design systems, Process (with a dropdown for the 5 phases), Activities.
Mobile
A toggle button that opens a WET overlay panel (#mb-pnl) with the same links in a stacked layout.

All menu links use absolute paths from /design/ (for example, /design/en/writing/index.html). The French equivalent is menu-include-fr.html with French labels and paths.

To add a new top-level nav item: Add a new <li> to the <ul class="list-inline menu" role="menubar"> in both menu-include-en.html and menu-include-fr.html. Changes take effect immediately on all pages.

3. assets/includes/footer-include-en.html — the footer

The footer fragment has two parts:

  1. Feedback call-to-action: A dark well linking to a Microsoft Forms survey for user feedback
  2. Community links: A “Join the community” link to a Teams channel and a “Contact us” mailto link (UCDGuide-GuideCAU@cra-arc.gc.ca), plus the Government of Canada wordmark

4. assets/css/ucdg-theme.css — the custom theme

This 289-line stylesheet does two things that define the site’s visual identity:

Menu-inverse theme
Overrides WET-BOEW’s default dark-background menu to create a light, transparent background with dark text. This is why the UCDG menu looks different from standard Canada.ca pages.
Step list component
The lst-stps and ld-zr classes that create numbered circle markers on ordered lists. This component is used on every activity page and many guidance pages.

Every content page loads this file via a relative path (for example, ../../assets/css/ucdg-theme.css). Getting the relative path wrong is one of the most common mistakes when creating new pages.
