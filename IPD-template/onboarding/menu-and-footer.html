How the menu and footer work

The menu and footer are not part of each page’s HTML. They are separate HTML fragment files that get loaded and injected at runtime by JavaScript.

On this page

How injection works

  1. Page loads and runs ucdg.js

    The script reads document.documentElement.lang to determine the language.

  2. Script selects the correct include files

    English: /design/assets/includes/menu-include-en.html and footer-include-en.html.
    French: /design/assets/includes/menu-include-fr.html and footer-include-fr.html.

  3. Script fetches each file via XMLHttpRequest

    Two sequential XHR calls load the HTML fragments from the server.

  4. Script injects the HTML into placeholder elements

    The menu HTML replaces the contents of <div id="menu-include">. The footer HTML replaces the contents of <div id="footer-include">.

Why XHR instead of a build system? This approach means changes to the menu or footer take effect on all pages immediately without rebuilding anything. The trade-off is a brief flash before the menu loads and a dependency on JavaScript being enabled.

Menu structure

The menu has two rendering modes in a single file:

Footer structure

The footer contains:

  • A feedback survey link (Microsoft Forms)
  • A Teams community link (“Join the community”)
  • A contact email (UCDGuide-GuideCAU@cra-arc.gc.ca)
  • The Government of Canada wordmark SVG

How to add a navigation item

  1. Open both menu include files

    Edit assets/includes/menu-include-en.html and menu-include-fr.html together.

  2. Find the desktop menu list

    Locate the <ul class="list-inline menu" role="menubar"> element.

  3. Add a new <li>

    Insert a new list item with an <a> link using an absolute path from /design/. For a submenu, nest a <ul class="sm list-unstyled"> inside the <li>.

  4. Update the mobile menu too

    Find the mobile overlay section (#mb-pnl) and add the same link there.

  5. Repeat in the other language file

    Add the French equivalent in menu-include-fr.html.
