Repository architecture

The UCDG is a static HTML site with no build system, no static site generator, and no server-side processing. Every page is hand-coded HTML served directly by GitHub Pages.

On this page

  • There is no build step

  • Directory map
  • Page count

There is no build step

There is no package.json, no Gemfile, no _config.yml, no Webpack, no Jekyll, no Hugo. You edit HTML files directly, commit, push, and the site updates. This means:

  • There is no automated linting, link checking, or HTML validation
  • Shared elements (menu, footer) are injected at runtime by JavaScript, not at build time
  • Every page manages its own relative paths to CSS, JS, and images
  • Changes to shared assets (CSS, JS, includes) take effect on all pages immediately

How deployment works

The site is deployed via GitHub Pages from the main branch. The custom domain is design.cra-arc.alpha.canada.ca. There is also a design-test branch for staging.

The site is served from the path /design/ (the repo name). This is important because ucdg.js uses absolute paths starting with /design/ to fetch the menu and footer includes. If the site were moved to a different path, the menu and footer injection would break.

All pages include <meta name="robots" content="noindex, nofollow">, telling search engines not to index the site. This is intentional — the UCDG is an internal resource.

Directory map

The repo has four top-level areas:

Top-level directory structure
PathWhat it contains
/assets/All shared static assets: CSS, JS, HTML includes, images, data files, and templates. This is the “engine room” of the site.
/en/All English content pages, organized by topic: writing, design-systems, process, activities, blog, changes, downloads.
/fr/All French content pages, mirroring the English structure with French directory names.
/ (root)The bilingual splash page (index.html), the 404 page, and README.md.
