Single page format

A single page presents all content on one screen, using “On this page” anchor navigation to help users jump between sections.

On this page

When to use a single page

Conditions for using a single page
Condition Example
The total content is manageable (a user can scan it without feeling overwhelmed)A simple claim with 4–5 short steps
The process is linear and straightforward with few decision pointsRequesting a remittance voucher
Users are likely to need most or all of the information in one sessionChecking the status of a return
The content does not have clearly distinct phases that a user would complete at different timesUpdating direct deposit information
The page can use “On this page” navigation to help users jump to what they needA reference page with several short sections

Indicators a single page is sufficient

  • The step-by-step instructions are under 8–10 steps
  • The eligibility criteria are simple (a short list, not a decision tree)
  • There is one primary user journey, not multiple branching paths
  • The total page length, while potentially long, is not confusing to navigate with anchor links

How to structure a single page

  1. Add “On this page” navigation

    If your page has more than 3 sections, add an “On this page” list immediately after the introductory text. Each item links to a section heading on the page.

  2. Follow the content model order

    Arrange your sections in the order defined in this guide: title, introductory text, before you start, eligibility, step-by-step instructions, and so on. Users benefit from consistent page structures across CRA services.

  3. Use expand/collapse for secondary content

    If some information is relevant only to a subset of users, use the expand/collapse (details/summary) pattern to keep the page scannable while still providing complete information.

  4. Test the page length

    If the page becomes difficult to navigate even with anchor links, reconsider whether the subway pattern would serve users better.
