Subway navigation

Used in: GST/HST (file-return), CAP (corporate instalments), T1 (filing sub-steps)

When to use: For linear multi-step journeys with 3–6 steps. The user follows a sequence where each step is its own page. The subway nav shows where you are in the process.

Landing page markup (parent page with section descriptions) 
<nav class="provisional gc-subway">
  <h2>Sections</h2>
  <dl>
    <dt><a href="...">Section Title</a></dt>
    <dd>Description of this section</dd>

    <dt><a href="...">Another Section</a></dt>
    <dd>Description of this section</dd>
  </dl>
</nav>
Child page markup (individual step with active indicator) 
<nav class="provisional gc-subway" data-sections-title="Sections"
     aria-label="Parent Section Title">
  <h1>Parent Section Title</h1>
  <ul>
    <li><a href="...">Step 1</a></li>
    <li><a class="active" href="...">Step 2 (current)</a></li>
    <li><a href="...">Step 3</a></li>
  </ul>
</nav>

<h1 property="name" id="wb-cont" class="gc-thickline">
  Step 2<span class="wb-inv"> &ndash; Parent Section Title</span>
</h1>

Important: Child pages must use layout: without-h1 in the YAML front matter because the H1 is rendered in the page body after the subway nav, not in the template header.

Doormats

Used in: CAP (index), GST/HST (topic page), EOL

When to use: On topic or landing pages to present 6–9 major sub-sections as linked cards. Each doormat has a linked H3 title plus a one-sentence description.

Doormat markup 
<section class="gc-srvinfo col-md-12">
  <div class="wb-eqht row">
    <div class="col-lg-4 col-md-6">
      <h3><a href="make-payment/index.html">Make a payment</a></h3>
      <p>How to make a full or partial payment to the CRA
         online, by mail, or in person</p>
    </div>
    <div class="col-lg-4 col-md-6">
      <h3><a href="...">Another Section</a></h3>
      <p>Brief description of this section</p>
    </div>
  </div>
</section>

The wb-eqht class makes all cards equal height. Use col-lg-4 col-md-6 for a 3-column desktop / 2-column tablet grid.

On This Page navigation

Used in: GST/HST (how-file), TFSA (extensively), EOL

When to use: On longer single pages with 3 or more H2 sections. Gives users a jump-to menu at the top of the page.

OTP navigation markup 
<nav role="navigation" class="gc-toc" aria-label="On this page">
  <h2 class="h3">On this page</h2>
  <ul>
    <li><a href="#h_1">First section heading</a></li>
    <li><a href="#h_2">Second section heading</a></li>
    <li><a href="#h_3">Third section heading</a></li>
  </ul>
</nav>

Each anchor must match the id attribute on the corresponding <h2> element in the page body.

Topic page with thick line

Used in: CAP, GST/HST

When to use: On the main topic page – the top-level entry point for a content area. The thick line signals "this is a major section."

Topic page heading markup 
<h1 property="name" id="wb-cont"
    class="provisional gc-thickline">
  Payments to the CRA
</h1>
<p class="pagetag">Find out how to send money to the CRA...</p>

The pagetag class provides a styled summary paragraph beneath the main heading.

Expand/hide (details/summary)

Used in: All COPs

When to use: To hide detailed content that not everyone needs behind a collapsible toggle. Common uses: additional examples, technical details, edge cases, definitions.

Expand/hide markup 
<details>
  <summary>Title of collapsible section</summary>
  <p>Content that is hidden by default and revealed
     when the user clicks the summary.</p>
</details>

This is native HTML5 – no additional classes needed. You can nest other elements (lists, tables, paragraphs) inside the <details> element.

Comparison tables

Used in: GST/HST (how-file – filing method comparison)

When to use: When users need to choose between multiple options and need to compare capabilities side by side.

Comparison table markup 
<table class="table wb-zebra">
  <caption>Compare filing methods</caption>
  <thead>
    <tr>
      <th scope="col">Feature</th>
      <th scope="col">Method A</th>
      <th scope="col">Method B</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Feature name</td>
      <td><span class="fas fa-check text-success">
          </span><span class="wb-inv">Yes</span></td>
      <td><span class="fas fa-times text-danger">
          </span><span class="wb-inv">No</span></td>
    </tr>
  </tbody>
</table>

The wb-zebra class adds alternating row colours. Use wb-inv for screen reader text alongside visual icons.

Conjunction lists

Used in: GST/HST (how-file – each filing method offers "Learn how" OR "File now")

When to use: When presenting two mutually exclusive options (learn about it vs. do it now, online vs. by mail).

Conjunction list markup (OR type) 
<ul class="cnjnctn-type-or cnjnctn-sm">
  <li class="cnjnctn-col">
    <h4>Learn how to file</h4>
    <p><a href="...">Using your CRA account</a></p>
  </li>
  <li class="cnjnctn-col">
    <h4>File now</h4>
    <p><a href="...">Sign in to your CRA account</a></p>
  </li>
</ul>

The cnjnctn-type-or class displays a visual "or" divider between the options. Use cnjnctn-type-and for both/and scenarios.

Font Awesome icon lists

Used in: GST/HST (contact section), CAP

When to use: For contact channels or action-oriented lists where icons help users quickly identify the channel type (online, phone, mail).

Icon list markup 
<ul class="fa-ul">
  <li>
    <span class="fa-li"><span class="fas fa-desktop">
    </span></span>
    <strong>Online -</strong> get account info
  </li>
  <li>
    <span class="fa-li"><span class="fas fa-phone-alt">
    </span></span>
    <strong>By phone -</strong> get answers
  </li>
  <li>
    <span class="fa-li"><span class="fas fa-envelope">
    </span></span>
    <strong>By mail -</strong> mail a document
  </li>
</ul>

Contextual sign-in button

Used in: GST/HST (every page via YAML front matter)

When to use: On pages where the user may want to sign in to their CRA account to complete the task described on the page.

YAML configuration for contextual sign-in 
auth:
  type: "contextual"
  label: "Sign in"
  labelExtended: "CRA sign in"
  link: "https://www.canada.ca/en/revenue-agency/services/
        e-services/cra-login-services.html"

This adds a "Sign in" button in the page header, styled according to GCWeb standards.
