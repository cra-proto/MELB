How the bilingual structure works

Every page exists in both English and French. The two languages are maintained as completely separate HTML files with hardcoded cross-links.

On this page

How the language toggle works

Each page has a language toggle in the header. It is a simple <a> link pointing to the other-language counterpart via a relative path. There is no JavaScript-based language switching.

The ucdg.js script uses the <html lang="en"> attribute to load the correct menu and footer includes. If the lang attribute is wrong, the page will load the wrong language’s navigation.

Known gaps

English has 71 pages; French has 67. The missing French pages are:

  • 4 subway sub-pattern demo pages (gc-subway-index.html, gc-subway-sub-content.html, gc-subway-sub-doormats.html, gc-subway-sub-mix.html)
  • The entire /downloads/ directory (3 downloadable template files)

Rules to follow

  • Always create the French counterpart when you create a new English page
  • Always update language toggle links in both directions
  • Use French directory names that match the topic in French, not transliterated English
  • The <html lang> attribute must be correct — ucdg.js depends on it
  • Shared data files (JSON) must include both _en and _fr fields
