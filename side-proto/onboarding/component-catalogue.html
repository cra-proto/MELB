How the component catalogue works

The GCWeb component catalogue at /en/design-systems/gcweb/index.html is the most complex page in the site. It uses a data-driven, filterable table rendered as a card grid.

On this page

How it works

The catalogue page uses several WET-BOEW features together:

  • wb-tables — makes the HTML table sortable and filterable
  • wb-actionmng and wb-doaction — power the sidebar filter buttons (Components, Templates, Design basics)
  • The table rows are rendered as visual cards using CSS, though the underlying structure is a <table>
  • Each row contains: a thumbnail image, a linked title, a description teaser, a type label (Components/Templates/Basics), and a category tag (GC core, CRA-specific, or CRA variant)

The data lives inline in the HTML <tbody>, not loaded from JSON. A separate file gcwebsysdata.json exists as an alternate data source but is not currently used by the live page.

How to add a new component

  1. Create the thumbnail images

    Create [component]-en.jpg and [component]-fr.jpg in assets/images/banners/gcweb/.

  2. Add a table row to the catalogue

    Open en/design-systems/gcweb/index.html and add a new <tr> to the <tbody> following the existing row pattern. Do the same in the French equivalent.

  3. Create the component detail page

    Create a new directory at en/design-systems/gcweb/[component]/index.html. Follow the standard component page structure: banner, intro, “On this page”, “When to use”, “How to implement”, “Design options”, accessibility, complementary components.

  4. Add a changelog entry

    Add an entry to assets/files/changes/whats_new_ucdg.json to announce the new component.

The JSON data file

en/design-systems/gcweb/gcwebsysdata.json contains the component catalogue as a JSON object. Each entry has: IMAGE (raw HTML), TITLE (raw HTML with link), TEASER, TYPE, CATEGORY, TAGS. This file exists as an alternate data source and may be used in future iterations to dynamically generate the catalogue.
