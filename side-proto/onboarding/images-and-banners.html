How images and banners work

The site uses responsive hero banner images on all hub and landing pages, and thumbnail images for the component catalogue.

On this page

  • Banner system
  • Required sizes
  • Component thumbnails
  • How banners load

Banner system

Each section of the site has its own hero banner image in assets/images/banners/. Banners are provided in four sizes for responsive loading. The naming convention is:

[section-name]-[breakpoint].png

For example: writing-991.png, writing-1400.png, writing-1700.png, writing-1900.png

Required sizes

-991990 × 800 px990 × 250 pxExtra-small screens

All banners are PNG with transparent backgrounds.

Existing banner sets: activities, blog, deliver, design, design-systems, discover, gcweb, initiate, monitor-refine, principles, process, ucdg-index, whatsnew, writing.

Component thumbnails

GCWeb component thumbnails live in assets/images/banners/gcweb/ and use JPG format:

  • Language-specific: [component]-en.jpg and [component]-fr.jpg
  • Language-neutral (some): [component].jpg (for visual-only components like charts, colour, icons)

How banners load

Hub pages use WET-BOEW’s data-bgimg-srcset attribute on a <div> to load the appropriate banner size based on viewport width. The syntax is:

data-bgimg-srcset="
  ../../assets/images/banners/writing-991.png 991w,
  ../../assets/images/banners/writing-1400.png 1400w,
  ../../assets/images/banners/writing-1700.png 1700w,
  ../../assets/images/banners/writing-1900.png 1900w"
