Gotchas for new contributors

Common gotchas
IssueWhat happensHow to avoid
Wrong relative path depth CSS, JS, and images fail to load. The page renders as unstyled HTML with no menu or footer. Always copy a page from the same directory depth. Count the ../ segments carefully.
Wrong <html lang> attribute The page loads the wrong language’s menu and footer. Double-check the lang attribute matches the page content.
Missing placeholder <div> Menu or footer does not appear on the page. Ensure both <div id="menu-include"> and <div id="footer-include"> exist.
Broken language toggle Clicking “Français” leads to a 404. Always create the French page first, then set the toggle links in both directions.
Editing menu includes Changes affect every page on the site instantly. Test menu changes locally before pushing. Edit both EN and FR files together.

Known inconsistencies

  • Font Awesome versions: Three different versions are loaded across pages (5.5.0, 5.8.1, 5.15.4). Use v5.15.4 for new pages.
  • Heading ID format: Some pages use h_1 (underscore), others use h-1 (hyphen). Use hyphens going forward.
  • French content gap: French is 4 pages behind English (subway examples and downloads directory).
  • GCWeb catalogue data source: A gcwebsysdata.json file exists but is unused. The catalogue currently uses inline HTML table rows.
  • Missing template: The templates README references a topic-1.html template that does not exist in the repo.
  • Commit messages: Many commits use generic messages like “Update index.html” with no description of what changed.

Suggestions for improvement

Add a .gitignore
The repo has no .gitignore. Add one to exclude .DS_Store, editor swap files, and other artifacts.
Pin WET-BOEW versions
JS and CSS are loaded from unpinned CDN URLs. A breaking change in WET-BOEW could affect the site without warning. Pin to a specific release.
Standardize Font Awesome
Pick one version (v5.15.4) and update all pages. Consider migrating to Font Awesome 6.
Add link checking
With no build system, broken links are only caught by manual testing. A simple CI workflow with a link checker (for example, htmlproofer or lychee) would catch broken links on every push.
Add a CONTRIBUTING.md
Document the page creation process, naming conventions, and review expectations so new contributors know what to do.
Write descriptive commit messages
Describe what changed and why, not just which file was touched.
Close the French gap
Translate the 4 missing French pages (subway examples) and add a /fr/telechargements/ directory with French template files.
Consider a static site generator
The manual boilerplate management is error-prone. A lightweight SSG (Eleventy, Hugo) could automate includes, breadcrumbs, and path resolution while keeping the hand-coded feel.
