Naming conventions and patterns

Consistency in naming makes the repo navigable. These are the conventions used across the codebase.

HTML IDs

Inconsistency: Content heading IDs use both underscore and hyphen conventions across different pages. Some pages use h_1, h_2, h_3; others use h-1, h-2. When creating new pages, use hyphens (h-1) for consistency.

Standard WET IDs are pre-defined: wb-sm (site menu), mb-pnl (mobile panel), wb-bc (breadcrumbs), wb-info (footer).

CSS classes

The site uses three layers of CSS classes:

  1. Bootstrap 3 utilities (from WET-BOEW): col-md-6, mrgn-tp-md, brdr-rds-0, btn-default, pull-left
  2. GCWeb theme classes: gc-thickline, gc-subway, gc-font-2019
  3. Custom UCDG classes: menu-inverse, lst-stps, ld-zr, stps-strpd, copyarea, ds-example, list-grid-hover, cnjnctn-*

Breadcrumbs

Breadcrumbs are built manually in every page using <nav id="wb-bc"> with an ordered list. They always start with “User-Centred Design Guide” linking to the home page, then each parent level.

All breadcrumb links use relative paths (unlike the menu, which uses absolute paths). This means breadcrumbs must be manually adjusted when a page moves to a different directory level.
